================================================================================
POTHOLE DETECTION APP - PROJECT INDEX
================================================================================

PROJECT OVERVIEW:
- Full-stack web application for real-time pothole detection
- Frontend: Next.js + React 19 + Tailwind CSS
- Backend: FastAPI + Uvicorn + PyTorch
- Model: YOLOv9+CBAM (your trained model)
- Real-time: WebSocket streaming + REST API
- Accuracy: 0.86 mAP@0.50

================================================================================
QUICK START (5 MINUTES)
================================================================================

1. Place your model:
   mkdir -p backend/models
   cp /path/to/best.pt backend/models/yolov9_cbam.pt

2. Start backend (Terminal 1):
   cd backend
   pip install -r requirements.txt
   python main.py

3. Start frontend (Terminal 2):
   npm install
   npm run dev

4. Open browser:
   http://localhost:3000

CONFIGURATION POINTS
================================================================================

Confidence Threshold:
  File: backend/main.py, line ~130
  Default: conf=0.5
  Change to: 0.6, 0.7, etc. for stricter detection

Camera FPS:
  File: app/camera/page.tsx, line ~115
  Default: 15 FPS
  Change to: 10, 20, etc. as needed

Image Input Size:
  File: backend/main.py, line ~128
  Default: imgsz=640
  Change to: 416, 512 for faster inference

GPU/CPU:
  File: backend/main.py, line ~127
  Default: device=0 (GPU)
  Change to: "cpu" for CPU only

================================================================================
TROUBLESHOOTING
================================================================================

"Model not found"
→ Check backend/models/yolov9_cbam.pt exists

"Connection refused" 
→ Is backend running? Check Terminal 1

"WebSocket connection failed"
→ Backend must run on localhost:8000

"CUDA out of memory"
→ Reduce imgsz to 416 or 512

"Camera access denied"
→ Allow permissions in browser settings

More help:
→ See QUICKSTART.md or SETUP.md troubleshooting sections
